Localhost passkey → card-cosigned Bitcoin signature

Proves the same flow as the Nuri PWA, fully local: a browser passkey on localhost produces a WebAuthn PRF, which is run through nuri-expo's exact derivation (HKDF(app:nuri.com|wallet|v1) → BIP86 m/86'/0'/0'/0/0) to get the client key. The physical card in your reader is the MuSig2 cosigner (musig2(client, card) + CSV recovery leaf). The result is a valid BIP340 signature for the Nuri Taproot address — no server, no sign.nuri.com.
